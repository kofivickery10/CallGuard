## 1. Purpose

This policy governs the secure use of the CallGuard platform within {{organization_name}}. It covers access, data handling, incident response, and regulatory alignment.

## 2. Scope

Applies to all personnel, contractors, and third parties with CallGuard access, covering:

- Call audio, transcripts, and scoring records.
- Compliance findings, breach records, and audit logs.
- Knowledge base content and API credentials.

## 3. Access Control

- Access is granted on the principle of **least privilege**.
- **Admin** role: compliance, IT, senior management only. Can create scorecards, manage agents, view all calls.
- **Member** role (agents): limited to viewing their own calls and scores.
- Accounts must be deactivated within 24 hours of a user leaving the organisation.
- Passwords must be unique, minimum 12 characters, not reused from other services.
- MFA is strongly recommended and should be enforced at the identity provider where possible.

## 4. Data Handling

- All audio, transcripts, and knowledge base files are **encrypted at rest** using AES-256-GCM.
- All API traffic uses **TLS 1.3** minimum.
- Personal data is **not downloaded to local machines** except where strictly necessary for a specific compliance review. Downloads must be deleted within 7 days.
- Transcripts may contain payment card or personal financial data - treat all transcripts as confidential (restricted).
- Share links generated for customer portal use are time-limited and must have expiry set to the minimum needed.

## 5. API Keys & Credentials

- API keys generated for external integrations (telephony, dialler) must be assigned individually named owners.
- Keys must be stored in a secrets manager, never in code, documents, or shared via email/chat.
- Revoke keys immediately when no longer needed.
- Review the list of active API keys quarterly.

## 6. Audit & Monitoring

- All admin actions (score changes, status updates, deletions) are logged in the CallGuard audit trail.
- The audit trail must be reviewed monthly by the Compliance Officer.
- Anomalous access patterns (bulk downloads, off-hours access) should be investigated within 24 hours.

## 7. Incident Response

A suspected data security incident involving CallGuard must be reported to {{dpo_email}} within 2 hours of detection. Incidents are triaged under the organisation's standard incident response plan, with particular attention to:

- Identification of affected data subjects.
- Determination of whether the incident is notifiable under UK GDPR Article 33 (72-hour regulator notification) and Article 34 (data subject notification).
- Root cause analysis within 7 days.

## 8. Regulatory Alignment

This policy supports obligations under:

- UK GDPR / Data Protection Act 2018
- FCA SM&CR (evidence of compliance oversight)
- FCA Consumer Duty (PS22/9) - evidencing good client outcomes
- COBS 9.5 / MiFID II record retention (7 years)

## 9. Policy Review

This policy is reviewed at least annually and reissued following any material change to CallGuard's functionality, vendor relationships, or regulatory environment.
